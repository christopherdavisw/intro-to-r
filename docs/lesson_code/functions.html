Function and Operator Cheat Sheet

This page hosts a function cheat sheet for each lesson.

I only describe the purpose of the function. If you would like to understand its arguments (inputs) and more details, use the help<function_name>) function or ?<function_name> or click on the link for each function.

Lesson 01 - Data Wrangling 1

Getting Started

  • help() : Opens up help documentation for packages and functions. ?<function_name> does the same.

R Basics.

  • \<- : Assign a value to a name.

Vectors.

  • c() : Generic function which combines its arguments. The default method combines its arguments to form a vector. All arguments are coerced to a common type which is the type of the returned value, and all attributes except names removed.

  • mean() : Generic function for the (trimmed) arithmetic mean.

  • sort() : Sort (or order) a vector or factor into ascending or descending order.

  • <from>:<to> : Generate regular sequences.

  • seq() : Generate regular sequences (with more options than :).

  • paste() : Concatenate vectors after converting to character.

  • nchar() : Takes a character vector as an argument and returns a vector whose elements contain the sizes of the corresponding elements of x.

  • length() : Get or set the length of vectors (including lists) and factors, and of any other R object for which a method has been defined.

  • [] : Operator that acts on vectors, matrices, arrays, and lists to extract or replace parts.

  • NA : NA is a logical constant of length ` which contains a missing value indicator.

  • is.na() : Indicates which elements of a vector are missing. Returns TRUE if missing, otherwise FALSE.

  • typeof() : Determined the (R internal) type or storage mode of any object.

  • is.numeric() : Determines if a vector is of a class numeric or not.

  • is.logical() : Determines if a vector is of type logical or not.

  • is.character() : Determines if a vector is of type character or not.

  • as.numeric() : Converts vector to numeric.

  • as.logical() : Converts vector to logical.

  • as.character() : Converts vector to character.

Arithmetic Operators

You can use arithmetic operators with numeric vectors as well as with single elements.

Here is a list of arithmetic operators:


Operator Name Example
+ Addition x + y
- Subtraction x - y
* Multiplication x * y
/ Division x / y
^ Exponent x ^ y
%% Modulus (Remainder from division) x %% y
&& Integer Division x&&y

Source: W3Schools

Comparison Operators

Comparison operators are used to compare two values.


Operator Name Example
== Equal x == y
!= Not equal x != y
> Greater than x > y
< Less than x < y
>= Greater than or equal to x >= y
<= Less than or equal to x <= y
%in% Have values within x %in% y


Logical Operators

Logical operators are used to combine conditional statements.


Operator Name Example
& AND operator. Returns TRUE if both elements are TRUE x == y & a == b
| OR operator. Returns TRUE if either elements are TRUE x == y | a == b
! NOT operator. Returns TRUE if the condition is not TRUE !(x == y


Packages and Libraries


  • install.packages() : Download and install packages from CRAN-like repositories or from local files.

  • library() : Attach add-on packages.

  • :: : Accessing exported and internal variables i.e. R objects (including lazy loaded datasets) in a namespace.


Data Frames and tibble


  • head() : Returns the first or last parts of a vector, matrix, table, data frame, or function.

  • tibble::as_tibble() : Turns an existing object, such as a data frame or matrix, into a so-called tibble, a data frame with class tbl_df.

  • class() : For objects which have a formal class, its name is returned. R posseses a simple generic function mechanism which can be used for an object-oriented style of programming.


dplyr Basics


  • dplyr::glimpse() : Prints information about a data frame or tibble to the console.

  • dplyr::mutate() : Creates new columns that are functions of existing variables. It can also modify (if the name is the same as an existing column) and delete columns (by setting their value to NULL).

  • dplyr::filter() : Used to subset a data frame, retaining all rows that satisfy your conidtions. To be retained, the row must produce a value of TRUE for all conditions. Note that when a condition evaluates to NA the row will be dropped, unlike base subsetting with [.

  • dplyr::select() : Select (and optionally rename) variables in a data frame, using a concise mini-language that makes it easy to refer to varaibles based on their name (e.g. <a>:<f> selects all columns from a on the left to f on the right) or type (e.g. where(is.numeric) selects all numeric columns).

  • tidyselect::everything() : Selects all variables. It is also useful in ocmbination with other tidyselect operators.

  • dplyr::arrange() : Orders the rows of a dta frame by the values of selected columns.

  • dplyr::count() : Lets you quickly count the unique values of one or more variables.

  • dplyr::group_by() : Most data operations are done on groups defined by variables. group_by() takes an existing tbl and converts it into a grouped tbl where operations are performed “by group”.

  • dplyr::ungroup() : Removes grouping.

  • dplyr::summarise() : Returns on row for each combination of grouping variables; if there are no grouping variables, the output will have a single row summarasing all ovservations in the input. It will contain one column for each grouping variable and one column for each of the summary statistics that you have specified. summarise() and summarize() are synonyms.

The pipe operator: |>

  • |> : Passes, or pipes, the result of the left-hand side expression to the right-hand side expression. If the right-hand side expression is a function call, then the left-hand-side is inserted as the first argument in the call.

Lesson 02 - Data Wrangling 2

Data Import and Export

  • readr::read_csv(): Read in .csv files.

  • readxl::read_excel(): Read in .xlsx or .xls files.

  • haven::read_sas() : Read in .sas7bdat or .sas7bcat files.

  • haven::read_xpt() : Read in SAS trasnfer files (.xpt).

  • haven::read_stata() : Read in .dta files.

  • readRDS() : Read in native .RDS files.

  • arrow::read_feather() : Read in .feather files.

  • arrow::read_parquet() : Read in .parquet files.

  • [data.table::fread()](https://www.rdocumentation.org/packages/data.table/versions/1.15.0/topics/fread) : Read in.csv` files faster.

  • readxl::excel_sheets() : Read in excel sheets form a .xlsx or .xls file.

  • readr::write_csv() : Write out .csv files.

  • haven::write_sas() : Write out .sas7bdat or .sas7bcat files.

  • haven::write_dta() : Write out .dta files.

  • saveRDS() : Save native .RDS files.

  • arrow::write_feather(): Write out .feather files.

  • arrow::write_parquet(): Write out .parqeut files.

  • janitor::clean_names() : Clean up messy variable names.

  • dplyr::if_else() : Takes in logical condition - if TRUE, output the value in the true argument, otherwise output the value in the false argument.

  • dplyr::case_when() : Takes in multiple logical conditions - basically use when you want to do multiple if/else statements. Each case is evaluated sequentially and the first match for each element determines the correspending value in the output vector.

  • stringr::str_c() : Concatenates a string.

  • stringr::str_length() : Provides the length of each element of a string.

  • dplyr::pull() : Extracts a clumn from a data frame.

  • stringr::str_detect() : Returns TRUE if pattern is matched in string. `FALSE`` otherwise.

  • stringr::str_pad() : Pads a string to be a specified width.

  • stringr::str_trim() : Removes whitespace from the start or end of a string.

  • stringr::str_sub() : Subset a string based on indices.

  • stringr::str_to_lower() : Converts strings to lower case (asdfjl).

  • stringr::str_to_upper() : Converts strings to upper case (ASDFJL).

  • stringr::str_to_title() : Converts strings to title case (This Is A Title).

  • stringr::str_to_sentence() : Converts strings to sentence case (This is a sentence.).

Attributes

  • attributes() : Returns the attributes of an R object.

  • names() : Returns the names attribute within an R object. Can also assign with names(x)<-.

  • class() : Returns the S3 object class of an R object.

  • factor() : Creates a factor variable.

  • as.Date() : Converts character or numeric to the Date class.

  • lubridate::ymd() : Flexibly convert data to Date, with format “year”, “month”, “day”.

  • lubridate::ydm() : Flexibly convert data to Date, with format “year”, “day”, “month”.

  • lubridate::myd() : Flexibly convert data to Date, with format “month”, “year”, “day”.

  • lubridate::mdy() : Flexibly convert data to Date, with format “month”, “day”, “year”.

  • lubridate::dmy() : Flexibly convert data to Date, with format “day”, “month”, “year”.

  • lubridate::dym() : Flexibly convert data to Date, with format “dau”, “year”, “month”.

  • lubridate::year() : Extract year from date.

  • lubridate::month() : Extract month from date.

  • lubridate::day() : Extract day from date.

  • lubridate::wday() : Extract weekday from date.

  • lubridate::quarter() : Extract quarter from date.

Reshaping Data

  • tidyr::pivot_longer(): Reshapes data longer.

  • tidyr::pivot_wider : Reshapes data wider.

Combining Data Frames

  • dplyr::bind_rows() : Appends data frames together.

  • dplyr::bind_cols() : Joins toether by column with no keys. Not recommended under most conditions.

  • dplyr::inner_join : Joins together by key, returns obsevation if key is present on both sides.

  • dplyr::full_join : Joins together by key, returns observation if key is present on either side.

  • dplyr::left_join : Joins together by key, returns observation if key is present on left side.

  • dplyr::right_join : Joins together by key, returns observation if key is present on right side.

  • dplyr::join_by() : Helper function for assigning by within join functions.

Lesson 03 - Data Visualization 1

The ggplot2 package

  • ggplot2::ggplot() : Initializes a ggplot object and creates an empty plot. Can also optionally set aesthetics that are inherited by later functions.

  • ggplot2::aes() : Aesthetic mappings describe how variables in the data are mapped to visual properties of geoms. Set in ggplot() and individaul layers.

  • ggplot2::geom_point() : Layer used to create scatterplots or points on a line graph.

  • ggplot2::geom_smooth() : Layer used to create a trend line over an existing plot.

  • ggplot2::scale_<axis>_continuous() : Scale used with continuous axis can set limits, breaks, and labels (among other things).

  • ggplot2::scale_<color or fill>_hue() : Maps each elvel to an evenly spaced hue on the color wheel.

  • ggplot2::facet_wrap() : Graphs multiple panels of data in a rectangular strcuture. Use a <var> syntax to generate panels by <var>.

  • ggplot2::geom_bar() : Layer that creates a bar plot with one varaible.

  • ggplot2::coord_flip() : Flips the X and Y axes.

  • ggplot2::theme() : Customize non-data components of plot.

  • ggplot2::element_rect() : Modify rectangular theme element.

  • ggplot2::element_blank() : Remove element from theme.

  • ggplot2::theme_minimal() : Complete theme that draws minimal lines and ticks.

Layers

  • ggplot2::geom_col() : Layer that creates a bar plot with two variables (minimum).

  • forcats::fct_reorder() : Reorders a factor on the order of another variable.

  • ggplot2::geom_segment() : Layer that draws a straight line between points. We can use it with geom_point() to create a lollipop chart.

  • ggplot2::geom_line() : Layer that creates a line graph.

  • ggplot2::geom_histogram() : Layer that creates a histogram.

  • ggplot2::geom_density() : Layer that creates a density plot.

  • ggplot2::geom_boxplot() : Layer that creates a box plot.

Aesthetics

No new functions! See the Aesthetic specifics ggplot2 vignette.

Scales

  • ggplot2::expansion() : Generate scale expansion vectors for the expand argument in a scale.

  • ggplot2::scale_y_reverse() : Transform y axis from high to low instead of low to high.

  • ggplot2::scale_<aesthetic>_manual() : Scale that allows you to specific mappings from levels in the data to aesthetic values (like colors).

  • ggplot2::scale_<color or fill>_brewer() : Maps values in data to an existing color pallete.

Text and Annotations

  • ggplot2::labs() : Modify axis, legend, and plot labels.

  • ggplot2::geom_text() : Layer that adds text from the data. No rectangle present behind text.

  • ggplot2::geom_label() : Layer that adds text from the data. Rectangle present behind text.

  • dplyr::slice() : Filtrs rows by their integer index locations.

  • ggplot2::annotate() : Add text (or another geom) to a graph not stemming from the data.

Themes

  • ggplot2::theme_gray() : This is the default ggplot2 complete theme that we’ve already seen.

  • ggplot2::theme_bw() : A dark on-light ggplot2 theme, which may work better for presentations displayed with a projecter.

  • ggplot2::theme_linedraw() : Another dark-on-light ggeplot2 theme, which may work better for presentations displayed with an old-timey projecter.

  • ggplot2::theme_light() : Light theme. Because of its light grey lines and axes, it focuses more attention on the data.

  • ggplot2::theme_dark() : Similar to theme_light() but the dark version. Good if you want to show colors with a dark background.

  • ggplot2::theme_minimal : A minimilastic theme. I quite like minimalistic themes as we focus more on the data.

  • ggplot2::theme_classic() : Also a minimalistic theme, with solid axes but no gridlines.

  • ggplot2::theme_void() : A completely empty theme. Good if you don’t want any axes or gridlines.

Functions and Iteration

Creating Functions

  • set.seed() : Sets a random number state such that randomly created numbers can be created in a reproducible manner.

  • rnorm() : Creates random numbers with n, mean, and standard deviation that follow a normal distribution.

  • function() : Defines new functions in the R language.

  • return() : returns an output within a function, exiting the function.

  • if() : checks if condition is TRUE, if so evaluates next statement.

  • else()) : after an if statement, this function will evaluate the proceeding expression if the preceding expression is FALSE

  • stop() : throws an error with an error message.

  • warning() : throws a warning with a warning message.

Working with Multiple Columns

  • dplyr::across() : applies a function across multiple columns in a data frame within a mutate() or summarise() function.

  • tidyselect::everything() : Matches all variables in a selection.

  • tidyselect::starts_with() : Matches columns that start with a specific prefix.

  • tidyselect::ends_with() : Matches columns that end with a specific suffix.

  • tidyselect::contains() : Matches columns that contain a specific string.

  • tidyselect::matches() : Matches a regular expression (see page 2 of the stringr cheat sheet)

  • <from:<to> : for selecting a range of consecutive variables. <from> and <to> can be numeric indicies of the position of the column or variable names (e.g.; carat:clarity).

  • ! : (in selection context), for taking all variables but the ones identified afterwards (e.g.: !carat:clarity would identify all variables but those between and including carat and clarity positionally in the dataframe).

  • c() : (in selection context) combine selections together.

  • tidyselect::all_of() and tidyselect::any_of() : Matches variable names given a character vector. These are the same except no error is thrown for any_of() if names don’t exist, but an error is thrown for all_of().

  • tidyselect::where() : applies a function to all varaibles and selects those for which the function returns TRUE.

  • dplyr::rename_with() : renames columns using a function.

  • dplyr::if_any() : apply the same function to a selection of columns and combine the result into a single logical vector. if_any() returns TRUE when the predicate is TRUE for any of the selected columns.

  • dplyr::if_all() : apply the same function to a selection of columns and combine the result into a single logical vector. if_all() returns TRUE when the predicate is TRUE for all of the selected columns.

  • list.files() : produces a character vector of the files in the specified directory.

  • purrr::map() : transform input by applying a function to each element of a list or atomic vector. Always returns a list.

  • purrr::list_rbind() : combines list elements into a data frame by row-binding them together.

  • readr::parse_number() : extracts the first number it finds, dropping any non numeric characters before and after.

  • rlang::set_names() : sets the names on an object and returns the object.

  • basename() : removes all of the path up to and including the last path separator.

  • purrr::walk() : calls a function but does not return an output.

  • purrr::walk2() : calls a function that does not return an output with two inputs of the same length.

  • purrr::map2() : calls a function that returns a list object with two inputs of the same length.
